2Rules before testing (things that look like bugs but aren't)

  1. One role per browser. Logging in as a second role ends the first session. To test two roles at once, use a second browser or an Incognito window.
  2. Switch roles by signing out: /api/auth/signout → Sign out → open the new role's login.
  3. Data is step-gated. A SOW only appears in the next role's queue after the previous role advances it (e.g. it reaches Glimmora's Commercial gate only after Enterprise approves Finance).
  4. Payments are simulated — no real money.
  5. OAuth (Google/Microsoft) buttons appear only on the Contributor login — intentional.

4The main flow — SOW → Delivery → Payment

The most important test. Runs across 4 roles in order. "Switch to X" = sign out, log in as X.

D1 Enterprise Create a SOW

Do: sidebar → SOW Workspace → New SOW → upload a PDF/DOC → pick a confidentiality level → Upload + extract → Submit for approval.

Expect: it shows the 5 approval stages (Security, Finance, Legal, Tenant-admin, Final), then "Submitted" with a SOW id; the SOW lists at stage Finance/Security.

D2 Enterprise Approve the internal gates

Do: open the SOW → on the approve page choose Approve → Approve stage, advancing through the enterprise-owned gates.

Expect: each gate flips to Approved; the SOW moves toward the Super-admin's queue.

D3 Super-admin Commercial approval + assign mentor

Do: switch to Super-admin → Commercial gate (/admin/sow) → open the SOW → tick the 3 checklist boxes → Assign a mentor (required) → comment → Approve Commercial.

Expect: Commercial = approved; records "Mentor assigned by Glimmora: …".

D4 Enterprise Final sign-off → Active

Do: switch to Enterprise → approve the remaining gates through Final sign-off.

Expect: SOW status becomes Approved / Active; all 5 stages show done.

D5 Enterprise Decompose into tasks

Do: sidebar → Decomposition → find the SOW → Decompose → open the plan → Submit for approval.

Expect: a plan of milestones + tasks is created and goes to Glimmora for pricing.

D6 Super-admin Work pricing (set contributor pay)

Do: Work pricing → open the plan (/admin/decomposition/[planId]) → per task, set Fixed ₹ or Hourly ₹/hr → Confirm → Save pricing. Click a task's View full details to see the brief + price.

Expect: the economics show Contributor payout · Glimmora margin · Client price · GST · Enterprise pays, plus a Budget comparison (actual SOW budget vs final enterprise price). An assigned task's price is locked.

D7 Contributor Express interest

Do: switch to Contributor → Opportunities → click a task to open its detail page (brief, acceptance criteria, files, deadline) → I'm interested.

Expect: price is shown up front (paid in full); button → "Interested · awaiting selection".

D8 Enterprise Select contributor + assign reviewer

Do: switch to Enterprise → open the task's Source & assign panel → pick from the interested pool → Select → assign a reviewer.

Expect: task → Assigned; reviewer recorded. (Enterprise never sees the contributor's pay.)

D9 Contributor Submit work

Do: switch to Contributor → Assigned → accept & start → do the work → Submit (summary + artifact link).

Expect: status "Submitted — awaiting mentor"; a version is saved (v1).

D10 Mentor Review gate (with rework loop)

Do: switch to Mentor → Queue → open the submission → read the Brief / Criteria / Evidence → either Request changes (test rework) then have the contributor resubmit (v2), or Accept (rate the rubric).

Expect: rework returns to the contributor with comments; accept advances to the reviewer. Both versions kept in history.

D11 Reviewer QA gate (final acceptance)

Do: switch to Reviewer → Queue → open the mentor-approved task → check the Task brief tab → Accept (rate) or Send back.

Expect: Accept → status "Accepted", an eligible payout is created; the milestone becomes payable.

D12 Glimmora / Enterprise Payout

Do: Glimmora requests the payout → Enterprise releases on Billing → Glimmora disburses to the contributor.

Expect: payout lifecycle eligible → requested → released → paid; the contributor sees it under Earnings; a "Payout sent" notification fires.

If D1–D12 all pass, the entire core platform works end-to-end.
