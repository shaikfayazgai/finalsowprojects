# Milestone v1.0: Frontend MVP

**Status:** ✅ SHIPPED 2026-02-27
**Phases:** 1–6
**Total Plans:** 29

## Overview

GlimmoraTeam is a 5-portal frontend monorepo (Women's, University, Enterprise, Mentor, Admin) sharing a Radix-based design system, TypeScript contracts, and Tailwind configuration. This milestone delivered the complete frontend in 6 phases: monorepo infrastructure and foundational design system first, then the full component library, then portals ordered by shared-component reuse (Women's + University together), governance complexity (Mentor, Enterprise), and finally the Admin Panel which depends on all entity types existing. All 162 v1 requirements are mapped. All data is mocked via MSW; Storybook is the backend handoff artifact.

- [x] 04-03: 3-panel review detail layout with evidence viewer and decision forms

**Details:**

## Milestone Summary

**Key Decisions:**

- Tailwind v4 CSS-first @theme (not v3) for simpler monorepo token sharing
- MSW v2 hybrid: types in @glimmora/types, handlers per-portal, shared factories extracted as needed
- DS-01..DS-10 in Phase 1 (not Phase 2) to enable canary validation with real styled components
- Radix UI unified package exports — use SlotPrimitive.Slot not bare Slot for asChild pattern
- Storybook 10 ESM-only: no addon-essentials, only @storybook/addon-a11y
- Component pattern: 'use client' per file, CVA variants, cn() utility, Radix primitives, forwardRef
- react-is override ^19.1.0 in root pnpm.overrides for recharts React 19 compatibility
- EvidenceViewer blind review enforced at TypeScript type level (contributor field absent from type)
- PoDL PDF: dynamic import only, never barrel-exported
- ResizablePanels v4: uses elementRef prop (not forwardRef), autoSaveId via wrapper hook
- OTPConfirmationDialog: shared component within enterprise-portal, reused for 4 OTP flows
- BulkPaymentRelease: useReactTable directly (not DataTable) for external rowSelection state
- SuperAdminGate: visible-but-locked pattern (not hidden/null) for standard admin users
- Audit log DataTable: structurally read-only with no action column by design

**Issues Resolved:**
- Phase 3 re-verification: 3 gaps closed (gradient KPI cards, student email+ID field, PDF lazy load)
- ResizablePanelGroup autoSaveId: v4 removed prop, added useDefaultLayout wrapper
- MSW pnpm.onlyBuiltDependencies for strict mode compatibility
- @glimmora/ui Label is a `<p>` element NOT `<label>` — use native HTML `<label>` for form associations
- `type:module` added to all packages for clean ESM under Node 25
- Port assignments: women=3001, university=3002, enterprise=3003, mentor=3004, admin=3005

**Issues Deferred (Tech Debt):**
- Enterprise evidence factory populates contributorId in mock data (medium — resolves at backend integration)
- Admin Panel missing podl_audit report type (medium — was not in v1 requirements)
- Font loading commented out in all portal layout.tsx files (low — needs font license files)
- OTPConfirmationDialog import path inconsistency (low — 3 direct vs 1 barrel)
- PoDL.verifiedByMentorId in contributor-facing type (low — backend should strip before sending)
- Admin settings MSW routes co-located in audit-log.ts handler (low — naming only)
