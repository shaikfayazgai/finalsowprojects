## Tech Debt Inventory

**Total: 6 items across 4 areas**

### Medium Priority

#### 1. Enterprise Evidence Factory Leaks `contributorId` in Mock Data
- **File:** `apps/enterprise-portal/src/lib/msw/factories/evidence.ts:12`
- **Issue:** Factory generates `contributorId: contributor-${randomId('c')}` in every Evidence object. The `toViewerEvidence()` adapter correctly strips it before UI render, so no identity ever appears in the UI. However, the mock factory models the wrong API contract. Production API must strip `contributorId` server-side before sending to enterprise clients.
- **Fix:** Remove `contributorId` from the enterprise evidence factory. The mentor portal's `ReviewEvidence` type correctly omits it — follow the same pattern.

#### 2. Admin Panel Missing PoDL Platform Audit Report
- **File:** `apps/admin-panel/src/app/(app)/reports/page.tsx`
- **Issue:** The 5 report types (`platform_overview`, `user_activity`, `project_delivery`, `financial`, `skill_growth`) do not include a PoDL credential ledger. Admin can see `podlCount` per user in user detail, but cannot query "show all PoDL credentials issued across a project/time range."
- **Fix:** Add `podl_audit` as a 6th report type with appropriate filters (date range, project, user type).

### Low Priority

#### 3. Admin Settings MSW Routes in Wrong Handler File
- **File:** `apps/admin-panel/src/lib/msw/handlers/audit-log.ts`
- **Issue:** Admin settings routes (`GET/PATCH/POST /api/admin/settings/admins`) live inside `audit-log.ts`. Routes ARE registered correctly (via `auditLogHandlers`). Naming confusion only.
- **Fix:** Move settings routes to a dedicated `settings.ts` handler file, register as `settingsHandlers` in `handlers/index.ts`.

#### 4. Font Loading Not Active in Portal Runtime
- **Files:** All `apps/*/src/app/layout.tsx`
- **Issue:** Miller Display and Avenir LT Std are commented out in portal layout files. Fonts load correctly in Storybook (via @font-face in preview.ts) but portal dev servers use fallback fonts.
- **Fix:** Obtain font license files and wire `next/font/local` in each portal's layout.tsx.

#### 5. OTPConfirmationDialog Import Inconsistency
- **Files:** `apps/enterprise-portal/src/components/sow/blueprint-approval.tsx` (barrel), vs. `payments/pending-approvals-table.tsx`, `projects/bulk-payment-release.tsx`, `projects/payment-release-card.tsx` (direct path)
- **Issue:** Three consumers use `../shared/otp-confirmation-dialog` directly; one uses `@/components/shared` barrel. All work.
- **Fix:** Standardize all four to the barrel import.

#### 6. `PoDL.verifiedByMentorId` in Contributor-Facing Type
- **File:** `packages/types/src/podl.ts`
- **Issue:** The `PoDL` interface includes `verifiedByMentorId: string`. PoDLCard correctly never renders it. But the raw PoDL response sent to contributor portals would expose this field in network payloads, potentially allowing contributors to know a specific mentor reviewed their work.
- **Fix:** Backend API should project-strip `verifiedByMentorId` before sending to contributor portals. MSW factory for contributor portals should not populate it.
